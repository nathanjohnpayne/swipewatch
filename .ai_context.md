# AI Context — Swipe Watch

## Project Name
Swipe Watch

## Project Type
Static web app (no build step, no framework, no server)

## Primary Languages
- JavaScript (Vanilla ES6+)
- HTML5
- CSS3

## Key Entry Points

| File | Purpose |
|------|---------|
| `index.html` | Full app shell — HTML structure, onboarding overlay, card container |
| `app.js` | All application logic — content pool, swipe mechanics, session management, coin bank, analytics |
| `styles.css` | All styling — animations, responsive breakpoints, card layouts |
| `firebase.json` | Firebase Hosting config — cache headers, SPA rewrites |
| `AGENTS.md` | AI agent instructions and behavioral rules |

## Directories With Non-Obvious Purpose

| Directory | Purpose |
|-----------|---------|
| `rules/` | Repository-level invariants enforced on all agents and contributors |
| `plans/` | Execution and rollout plans for features or migrations |
| `specs/` | Intended behavior definitions (acceptance criteria, feature specs) |
| `tests/` | Placeholder — no automated tests currently exist (see below) |
| `functions/` | Placeholder — no serverless functions currently exist (see below) |
| `docs/` | Extended documentation beyond README/AGENTS.md |
| `scripts/ci/` | CI enforcement checks validating compliance with the AI Agent Tooling Standard |

## Directories Intentionally Absent

| Directory | Justification |
|-----------|--------------|
| `dist/` | No build step exists. Static files are deployed directly from source. If a build step is introduced, create this directory and update `.gitignore`. |
| `src/` | All source files (`index.html`, `app.js`, `styles.css`) live at the repository root. No subdirectory structure is needed at this project scale. |

## Standard Compliance Notes

- **`dist/` absent:** Intentional. Static site with no build output. Documented above.
- **`src/` absent:** Intentional. Source lives at root. Documented above.
- **`tests/` empty:** No automated test framework is in use. `tests/` exists as a placeholder per the standard. Add tests when a testing approach is established.
- **`functions/` empty:** No serverless or cloud functions. Exists as a placeholder. Add if backend functions are introduced.
- **`scripts/ci/` wired to CI:** CI enforcement scripts run automatically on push/PR via `.github/workflows/repo_lint.yml`. They can also be run locally before commits.
- **`.cursor/rules/deploy.mdc`:** A Cursor `.mdc` rules file providing deploy context to the Cursor agent. This is valid Cursor configuration (not a plain `.md` instruction file). Content mirrors `DEPLOYMENT.md`.

## Related Repositories

- **`mergepath`** (sibling directory) — Canonical template per the AI Agent Tooling Standard. Contains the `gcloud` wrapper, `op-firebase-deploy` / `op-firebase-setup` scripts, the 1Password-backed impersonation deployment guide, and the full standard definition. Reference when creating new projects or when deployment questions extend beyond this repo.
- **Deploy-auth invariant** — This repo intentionally uses the shared 1Password-backed `Private/GCP ADC` source credential for routine human `gcloud` and deploy work. Do not migrate it back to ADC-first or deploy-key auth without explicit human approval.
